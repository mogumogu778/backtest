# 日本株バックテストツール

日本株（東証上場銘柄）を対象に、テクニカル戦略のバックテストをブラウザ上でインタラクティブに検証できるツールです。

---

## 機能概要

- 東証上場銘柄のOHLCVデータをリアルタイム取得（Yahoo Finance経由）
- 4種類のテクニカル戦略でバックテストを実行
- シグナルは**当日終値で発生→翌日始値で約定**（ルックアヘッド・バイアスなし）
- 手数料・スリッページを含んだ現実的なコスト計算
- インサンプル／アウト・オブ・サンプル分割による過剰最適化の検出
- 資産推移・売買シグナル・トレード履歴をインタラクティブに可視化

---

## 対応戦略

| 戦略 | 概要 |
|------|------|
| 移動平均クロス | 短期MAが長期MAを上抜けで買い、下抜けで売り |
| RSI | 売られすぎ閾値を下抜けで買い、買われすぎ閾値を上抜けで売り（Wilder EMA法） |
| ボリンジャーバンド | 下限バンド割れで買い、上限バンド超えで売り |
| MACD | MACDラインがシグナルラインを上抜けで買い、下抜けで売り |

## セットアップ

```bash
pip install -r requirements.txt
```


## 使い方

### 1. 銘柄・期間の設定

サイドバーの「設定」欄で銘柄コードと検証期間を入力します。

- **銘柄コード**: 東証銘柄は末尾に `.T` を付けてください
  - 例: `7203.T`（トヨタ）、`9984.T`（ソフトバンクG）、`1321.T`（日経225 ETF）
- **開始日 / 終了日**: 検証したい期間を選択します（デフォルトは直近3年）

### 2. 戦略とパラメータの設定

「戦略」ドロップダウンから戦略を選択すると、対応するパラメータが表示されます。

| 戦略 | 設定項目 |
|------|---------|
| 移動平均クロス | 短期MA期間（日）、長期MA期間（日） |
| RSI | RSI期間（日）、売られすぎ閾値、買われすぎ閾値 |
| ボリンジャーバンド | 期間（日）、標準偏差の倍数 |
| MACD | 短期EMA、長期EMA、シグナル期間 |

### 3. 資金・コストの設定

| 項目 | 説明 | デフォルト |
|------|------|-----------|
| 初期資金 | バックテスト開始時の資金（円） | 1,000,000円 |
| 手数料率 | 売買時にかかる手数料（%） | 0.10% |
| スリッページ | 注文価格と約定価格のズレ（%）。買いは高く、売りは安く約定する想定 | 0.05% |

### 4. バックテスト実行

「バックテスト実行」ボタンを押すと、結果が画面に表示されます。

## 結果の見方

### パフォーマンス指標（3行表示）

| 指標 | 説明 |
|------|------|
| 総リターン | 期間全体の累積リターン。Buy & Holdとの差分も表示 |
| 年率リターン | 年換算したリターン |
| Buy & Hold リターン | 同期間・同銘柄をただ保有し続けた場合のリターン。戦略の優位性の基準 |
| シャープレシオ | リスク当たりのリターン効率。1.0以上で優秀、2.0以上で極めて優秀 |
| 最大ドローダウン | ピークからの最大下落率。精神的に許容できる水準か確認する |
| カルマー比率 | 年率リターン ÷ 最大ドローダウン。ドローダウン当たりの効率 |
| Profit Factor | 総利益 ÷ 総損失。1.0未満は損失超過、∞はノーロス |
| 勝率 | 利益で終わったトレードの割合 |
| 平均リターン/トレード | 1トレード当たりの平均損益率 |

### タブ表示

- **株価チャート**: ローソク足と戦略のテクニカル指標、売買シグナル（▲買い・▽売り）
- **資産推移**: 戦略の資産推移と Buy & Hold の比較チャート
- **トレード履歴**: 全売買の日付・価格・損益一覧

## アウト・オブ・サンプル（OOS）検証

過剰最適化（カーブフィッティング）を検出する機能です。

### 有効化の方法

サイドバーの「アウト・オブ・サンプル検証」欄で「OOS検証を有効化」にチェックを入れると、テスト期間の割合スライダーが表示されます。

### 仕組み

データを2つの期間に分割し、**同じパラメータで別々にバックテスト**します。

```
全データ期間
├── インサンプル（前半）  ← パラメータ調整に使った期間
└── アウト・オブ・サンプル（後半）← 未知データでの検証期間
```

### 結果の解釈

- **ISとOOSの成績が近い**: パラメータが汎用的で信頼性が高い
- **OOSの成績が極端に悪い**: 過去データに過剰適合している可能性が高い

> パラメータを変えながら何度も実行し、OOSで成績が良いパラメータを選ぶ行為は、それ自体がOOSデータへの過剰最適化になります。ISとOOSの分割比率を固定したうえで、最初に決めたパラメータのOOS成績を一度だけ確認することが正しい使い方です。

## ディレクトリ構成

```
claude-backtest/
├── app.py                        # Streamlit アプリ本体
├── requirements.txt
├── data/
│   └── fetcher.py                # Yahoo Finance からのデータ取得
└── backtest/
    ├── engine.py                 # バックテストエンジン（約定・損益計算）
    ├── metrics.py                # パフォーマンス指標の計算
    └── strategies/
        ├── base.py               # 戦略の基底クラス
        ├── ma_cross.py           # 移動平均クロス
        ├── rsi.py                # RSI
        ├── bollinger.py          # ボリンジャーバンド
        └── macd.py               # MACD
```

## 注意事項

- 本ツールは学習・研究目的のバックテスト環境です。投資判断の根拠として使用しないでください。
- バックテストの成績は過去データに基づくものであり、将来の運用成績を保証するものではありません。
- データはYahoo Finance（yfinance）経由で取得します。データの正確性・完全性は保証されません。
